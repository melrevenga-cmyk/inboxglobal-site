// Severity framework

Five levels. One shared
operating language.

Level Classification Trigger (any one) First Action Response Window
SEV-1 Critical — Domain Down Blacklisted on major list, account suspended, reputation "Bad" 7+ days, inbox placement <10% Stop all sending immediately. Notify leadership. Open SEV-1 playbook. Begin within 1 hour
SEV-2 Severe — Reputation Compromised Reputation "Low," placement 10–40%, complaint rate >0.3%, bounce spike >5% Pause all campaigns within 2 hours. Run diagnostic. Open SEV-2 playbook. Begin within 4 hours
SEV-3 Moderate — Active Degradation Bounce 3–5%, complaint 0.1–0.3%, placement 40–65%, reputation "Medium," 1 inbox showing issues Pause affected inboxes. Reduce volume 50%. Open SEV-3 playbook. Begin same day
SEV-4 Warning — Drift Detected Auth misconfigured, sending limits exceeded, warmup lapsed, monitoring gap 14+ days Document the drift. Fix configuration issues within 48 hours. Fix within 48 hours
SEV-5 Hygiene — Maintenance Required List stale, docs outdated, verification overdue, quarterly review missed Schedule fix. Assign owner. Document what needs updating. Resolve within 2 weeks
// What's inside

Seven sections. One
complete workspace.

A

Welcome + How to Use

Onboarding guide for the workspace. What each section is for, when to use it, and how to hand it off to a new team member without explanation.

B

Severity Framework Database

Full Notion database with table and gallery views. All five severity levels with trigger conditions, pipeline impact, response windows, and links to matching playbooks.

C

Diagnostic Runbook

Eight-step structured triage — from "something feels off" to a confirmed severity level in 15–20 minutes. Toggle blocks for each step. Built for use during a live incident.

D

Five Incident Response Playbooks

One per severity level. Time-boxed actions (Hour 1, Hours 2–4, End of Day), all as checkboxes. SEV-1 includes delisting process, recover vs. retire framework, and a stakeholder communication template.

E

Four Recovery Timeline Templates

90-day full rebuild (SEV-1), 60-day reputation recovery (SEV-2), 30-day stabilization (SEV-3), and a domain retirement and fresh-start checklist. Structured as project trackers with week-by-week phases.

F

Enforcement Rules

Sending limits, list quality gates, monitoring cadence, escalation triggers, and change control — defined in tables your team can implement immediately and review monthly.

G

Quick Reference Card

One-page cheat sheet: severity levels at a glance, sending limits, and a five-step "when in doubt" protocol. Built to be bookmarked. The first place operators go when something breaks.
